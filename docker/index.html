git-版本控制

为什么要控制版本：回溯

  • RCS
  • SVN：集中式
  • Git：分布式

git-核心思想

  • 分布式与多中心
  • 本地执行
  • 记录快照
  • 只添加数据

git-三种状态

  • staged：已暂存
  • modified：已修改
  • committed：已提交

git-基本操作

  • 获取代码-git pull
  • 在工作目录中修改文件
  • 暂存文件-git add
  • 提交更新-git commit
  • 提交到远程-git push

git-工作流

  • 管理人员分配功能分支
  • 开发人员获取分支代码
  • 在本地分支上进行开发
  • 完成发出pull request

docker-容器技术

  • 虚拟化技术：隔离的环境
  • 虚拟机：内核空间的模拟
  • 容器：用户空间的模拟

docker-核心概念

  • Image-镜像
  • Container-容器
  • Volume-数据
  • Registry-仓库

docker-基本操作

  • 获得：pull,load,import
  • 创建：run与-d, -it, -p, -v
  • 运行：start, stop
  • 操作：nsenter,attach,exec

docker-工作流

  • 修改代码
  • 运行容器编译
